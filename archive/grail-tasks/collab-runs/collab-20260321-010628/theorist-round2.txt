## THEOREM E: The Integer Weight Plotkin Bound

$$\sum_{s \in \mathcal{C}} \text{wt}_{\mathbb{Z}}(s) = |\mathcal{C}| \cdot m \cdot \frac{p-1}{2}$$

The average value is $\frac{1}{p^d} \sum_{a=0}^{p-1} p^{d-1} \cdot a = \frac{p-1}{2}$.

If the form is zero for $k$ coordinates: $\sum \text{wt}_{\mathbb{Z}} = p^d \cdot (m-k) \cdot \frac{p-1}{2}$.

For the dual of a full-rank generator matrix ($V$ has rank $r+1$): no coordinate is identically zero on $\mathcal{C}$ (since each column $v_i \neq 0$, wait — actually $s_i = 0$ on all of $\mathcal{C}$ means the $i$-th standard basis vector is in $\mathcal{C}^\perp = \text{rowspan}(V)$, which means $e_i$ is a linear combination of the rows of $V$. This could happen, but for our specific $V$ it means vector $v_i$ has a specific structure).

In the generic case (no zero coordinates): $\sum_{s \in \mathcal{C}} \text{wt}_{\mathbb{Z}}(s) = p^d \cdot m(p-1)/2$.

Average over nonzero codewords: $\frac{p^d \cdot m(p-1)/2}{p^d - 1} \approx m(p-1)/2$.

**Plotkin-type bound.** If every nonzero codeword has integer weight $> p$, then:
$$m(p-1)/2 \approx \text{average wt}_\mathbb{Z} > p$$
$$m > \frac{2p}{p-1} = 2 + \frac{2}{p-1}$$

This gives $m \geq 3$. But we have $m = 2\alpha(r)+2 \gg 3$. The bound is trivially satisfied and gives **no contradiction**.

The Plotkin bound is too weak because the integer weight concentrates around $m(p-1)/2$, which is much larger than $p$ for large $m$. The constraint "weight $> p$" only removes a tiny fraction of codewords.

**Better approach: use the COMPLEMENT weight.**

Define $\bar{s}_i = p - 1 - s_i$ (not $p - s_i$). Then $\text{wt}_\mathbb{Z}(\bar{s}) = m(p-1) - \text{wt}_\mathbb{Z}(s)$. The map $s \mapsto \bar{s}$ is NOT linear (it's affine: $\bar{s} = (p-1,\ldots,p-1) - s$).

The codewords of weight $\leq p$ are those satisfying $\sum s_i \leq p$, meaning $s$ has small support (most $s_i = 0$) or small nonzero values.

**The number of vectors in $\mathbb{F}_p^m$ with integer weight $\leq p$:** This is the number of $(s_1,\ldots,s_m) \in \{0,\ldots,p-1\}^m$ with $\sum s_i \leq p$. Each $s_i \geq 0$ and $\sum s_i \leq p$: by stars and bars (since each $s_i < p$ is non-binding when $\sum \leq p$ and $m \geq 2$), this is $\binom{m+p}{p}$.

The fraction of such vectors: $\binom{m+p}{p} / p^m$.

For $m = 18, p = 5$: $\binom{23}{5}/5^{18} = 33649/5^{18} \approx 33649 / 3.8 \times 10^{12} \approx 10^{-8}$. Very small!

For the code to avoid ALL of these (except $s = 0$), we need: $\mathcal{C} \cap B_p = \{0\}$ where $B_p = \{s : \text{wt}_\mathbb{Z}(s) \leq p\}$.

**Gilbert-Varshamov-type bound.** If $|\mathcal{C}| > p^m / |B_p|$, then by pigeonhole, $\mathcal{C}$ must intersect some translate of $B_p$, but NOT necessarily $B_p$ itself. This doesn't directly help.

**Direct counting.** $|\mathcal{C} \setminus \{0\}| = p^d - 1$. $|B_p \setminus \{0\}| = \binom{m+p}{p} - 1$.

If $\mathcal{C} \cap B_p = \{0\}$: the $p^d$ elements of $\mathcal{C}$ must all lie in $\{s : \text{wt}_\mathbb{Z}(s) > p\} \cup \{0\}$, which has $p^m - \binom{m+p}{p} + 1$ elements.

For this to be possible: $p^d \leq p^m - \binom{m+p}{p} + 1$. This is always true for large $p$ since $\binom{m+p}{p}$ grows polynomially while $p^m$ grows exponentially. **No contradiction this way either.**

### Insight: Integer Weight is the WRONG metric

The coding theory approach via Plotkin/GV bounds doesn't give sharp enough results because:
1. The integer weight ball $B_p$ is exponentially small compared to $\mathbb{F}_p^m$
2. The kernel code is exponentially large
3. But codes CAN avoid exponentially small sets

**The ZSF condition is not about a code avoiding a ball — it's about a very specific algebraic structure (the kernel of a matrix) avoiding a very specific combinatorial set (integer weight $\leq p$).** Generic coding bounds can't capture this.

### New Direction: The Inductive Weight Decomposition

**Theorem F (Cross-Fiber Reduction to Weight-Constrained Sumset).** Consider $S = F_1 \cup F_c$ with $|F_1| = |F_c| = \alpha(r)+1$, $c \neq 0, 1$. WLOG $c \not\equiv -1 \pmod{p}$ (the case $c = p-1$ is handled separately).

**Since $c \neq 1$, the split is uniquely determined by $w$.** As $w$ ranges over $\{1,\ldots,p-1\}$, the pair $(W_1, W_2) \pmod{p}$ cycles through all pairs with $W_1 + cW_2 \equiv 0$.
