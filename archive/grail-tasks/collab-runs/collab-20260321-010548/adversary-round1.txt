## ADVERSARY REPORT: Round 1

### Attack 1: Pair-Degenerate Weight Bound (NEW, PRECISE)

**Claim:** For any collinear triple with a pair-degenerate dependency (α, β, γ) — meaning some pair sums to 0 — the minimum weight over all rescalings μ ∈ F_p* is **exactly p+1**.

**Proof:** WLOG γ = 1. Pair-degenerate means α = -1, β = -1, or α + β = 0. Take the case α = -1 (others symmetric). Weight under rescaling by μ:

> W(μ) = ⟨-μ⟩ + ⟨μβ⟩ + ⟨μ⟩ = (p - ⟨μ⟩) + ⟨μβ⟩ + ⟨μ⟩ = **p + ⟨μβ⟩ ≥ p + 1**

Similarly for β = -1: W(μ) = ⟨μα⟩ + (p - ⟨μ⟩) + ⟨μ⟩ = p + ⟨μα⟩ ≥ p + 1.

For α = -β: W(μ) = ⟨-μβ⟩ + ⟨μβ⟩ + ⟨μ⟩ = p + ⟨μ⟩ ≥ p + 1.

**Conclusion:** Pair-degenerate triples are DEAD for weight ≤ p. The rank-3 theorem is the ONLY support-3 tool, and it requires non-degeneracy.

### Attack 2: Lemma 2 from Post-Mortem is Trivially True (not useful)

The post-mortem's Lemma 2 ("triple-sum-zero gives weight exactly p") follows from one line:

> **S(μ) + S(p-μ) = 3p** for any triple with α+β+γ = 0.

Because ⟨x⟩ + ⟨p-x⟩ = p for all x ∈ F_p*. So if S(μ) = 2p, then S(p-μ) = p. Weight = p ≤ p. ✓

This is correct but uninteresting — it gives weight EXACTLY p, never < p. The sum-zero case is a boundary case, not a tool for progress.

### Attack 3: Fiber Induction Is Provably Incomplete (CRITICAL)

The Pigeonhole ratio for fiber induction is:

> (α(r) + 2) / (α(r-1) + 1) = (2α(r-1) + 3) / (α(r-1) + 1) → **2 as r → ∞**

For ANY r ≥ 4 and ANY p ≥ 5: this ratio is strictly less than 3. So with **≥ 3 nonempty fibers**, no fiber is guaranteed to have α(r-1)+1 vectors. Within-fiber induction via the η or s constants NEVER fires.

Explicitly: at rank 5 with 36 vectors across 3 fibers (12+12+12), each fiber has 12 projections in F_p^4. Need 18 for η(C_p^4). Gap of 6. No amount of cleverness with η/s fixes this.

**The two-fiber bound works** (I verified: Pigeonhole gives max(a,b) ≥ α(r)+1 when there are exactly 2 fibers, which suffices for η in fiber 0 or s in fiber c). But it's a CLIFF — going from 2 to 3 fibers destroys the argument entirely.

### Attack 4: Cap Obstruction Kills Support-3 for Large p (CRITICAL)

The maximum cap size in PG(n, F_p) (set of points with no three collinear) satisfies:

- PG(3, F_p): cap size ≤ p² + 1 (ovoid bound, achieved for all odd p)
- PG(4, F_p): cap size ~ p² (Bose, Segre)

At rank 4 (PG(3, F_p)): we need α(4)+2 = 19 non-proportional points. For p ≥ 5: p² + 1 ≥ 26 > 19. So **18 points CAN be placed with no three collinear** in PG(3, F_p) for all p ≥ 5.

At rank 5 (PG(4, F_p)): need 37 points. Cap bound ~p². For p ≥ 7: 49 > 37. Again, all points in general position is possible.

**When no three are collinear, there are NO support-3 zero-sums at all.** The rank-3 theorem is completely useless. Any proof must find zero-sums of support ≥ 4.

Combined with Attack 1: even if some triples ARE collinear, if they're all pair-degenerate, support-3 is still dead.

### Attack 5: The Support-4 Problem Is Fundamentally Harder

At p = 5, the 94 residual cases (out of 607) were killed by support-4 zero-sums of weight 4 < p = 5. These are:
- 4 vectors summing to zero with all coefficients = 1

This is the "4-sum-free set" problem: can a set of 18 points in F_p^4 avoid v₁+v₂+v₃+v₄ = 0?

For weight ≤ p at support 4: coefficients (s₁,s₂,s₃,s₄) with each s_i ∈ {1,...,p-1} and Σs_i ≤ p. Since each s_i ≥ 1, we need Σs_i ∈ {4, 5, ..., p}. The number of such coefficient tuples grows polynomially in p. But the algebraic conditions (Σs_i·v_i = 0) are HARD to analyze without the character-theoretic machinery that's been declared dead (Dead End #1).

**No existing tool handles support ≥ 4.** The rank-3 theorem handles support 3. Character theory for support 4 is dead (two energy levels, not one). The polynomial method (CW) is underdetermined. This is the core gap.

### Attack 6: Stress-Testing the Inductive Approach (Approach 1)

The proposed inductive upper bound mirrors the lower bound: assume η(C_p^r) ≤ α(r)(p-1)+1, prove η(C_p^{r+1}) ≤ (2α(r)+1)(p-1)+1.

**This approach CANNOT work by fiber induction alone** (Attack 3). It requires a cross-fiber argument. What would that look like?

A cross-fiber zero-sum involves vectors from ≥ 2 fibers. The fiber constraint: Σ_i s_i·c_i ≡ 0 (mod p), where c_i is the fiber of vector i. This is ONE linear equation in the coefficients. Combined with the r equations from the projection (Σ s_i · proj(v_i) = 0), we have r+1 equations — the original zero-sum condition.

So there's no "decomposition gain" from fibers in the multi-fiber case. The fiber structure doesn't simplify the problem; it just reparametrizes it.

This would close the gap. But I suspect this is FALSE for support-3 and the real action is at support 4+.
