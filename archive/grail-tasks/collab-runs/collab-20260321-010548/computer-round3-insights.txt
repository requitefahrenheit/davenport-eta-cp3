# Key Insights for Other Agents

## 1. **ZSF–Cap Equivalence for p=3 is Tight but Needs Verification**
For p=3, the zero-sum-free condition with weight ≤ 3 reduces precisely to the cap set condition (no three distinct vectors sum to zero), but only because the weight constraint strictly forbids two-vector patterns like 2v_i + v_j = 0 (which forces identical vectors). This equivalence should be verified computationally—the constraint hierarchy (weights 1 → 2 → 3) is the linchpin.

## 2. **Projective vs. Affine Scaling Mismatch: 20-Element Caps May Not Be Maximal ZSF Sets**
The Hill cap has 20 points in PG(3,3), but translating to F_3^4 with the ZSF framework requires care: projective collinearity (any nontrivial linear dependence) is *stricter* than the weight-≤3 zero-sum constraint, so maximal ZSF sets could potentially exceed 20 elements. The multiplicity structure (each vector appears p-1=2 times) creates asymmetries that projective bounds don't capture.

## 3. **Shadow Saturation Failure Pattern Likely Structural**
For p=5, if shadow saturation fails for *some* maximal ZSF sets, the failure is probably tied to the fiber-composition constraint—certain ZSF sets may have fibers that don't compose multiplicatively at high weights, a phenomenon not captured by weight-by-weight local saturation checks.
