## ATTACK 2: Projection Strategy Has a Provable Arithmetic Gap

The projection strategy (Cortex entries `19d0b14f4fa`, `19d091a9035`): project α(r)+1 vectors along one vector v_k, apply the inductive hypothesis in the quotient F_p^{r-1}, lift back.

**The lift always gives c ≠ 0.** Here's why:

A quotient zero-sum Σs_i·π(v_i) = 0 lifts to Σs_i·v_i = c·v_k in F_p^r. If c = 0, we have Σs_i·v_i = 0 at weight ≤ p — a zero-sum among the ORIGINAL vectors, contradicting the ZSF assumption.

**Therefore c ≠ 0 for EVERY quotient zero-sum.** This is not a rare failure; it's a theorem.

To complete the zero-sum, set s_k = p−c. Total weight becomes (quotient weight) + (p−c). For weight ≤ p, we need:

**(quotient weight) ≤ c**

But the inductive hypothesis only guarantees quotient weight ≤ p. And c ∈ {1,...,p-1}. When quotient weight = p (the maximum the theorem gives), the condition c ≥ p is IMPOSSIBLE.

**The projection strategy therefore requires a STRICT STRENGTHENING of the inductive hypothesis**: not just "zero-sums at weight ≤ p" but "zero-sums at weight ≤ c" for a geometry-dependent c < p.

The computer verified this works at p=5,7 for rank 4 (Cortex `19d0b14f4fa`). The minimum lifted weight distribution shows weights 3,4,5 at p=5 and 3,4,5,6,7 at p=7 — so low-weight quotient zero-sums exist in practice. But **no theorem guarantees their existence in general**.

## ATTACK 3: Fiber Induction Has Zero Surplus

The Weight Flexibility / Cauchy-Davenport approach (Cortex `19d0f99be96`):

N = 2α(r-1)+2 vectors across k ≥ 3 fibers. Largest fiber has at most ⌈N/2⌉ ≈ α(r-1)+1 vectors. **Surplus over α(r-1) ≤ 1.**

Weight Flexibility at surplus 1: produces f(1) distinct achievable weights. Even optimistically, f(1) ≈ O(1).

Cauchy-Davenport combining two fibers: |W₁ + W₂| ≥ |W₁| + |W₂| − 1 ≈ 1+1−1 = 1.

**One achievable weight gives no useful information.** The cross-fiber combination needs weights to be in complementary ranges ({1,...,c} and {1,...,p−c} for some c), requiring Ω(p) achievable weights per fiber. Surplus 1 gives O(1). Fatal arithmetic mismatch.

This is the same as my Attack 3 from last round, but now I'm emphasizing the QUANTITATIVE failure: the surplus is bounded by a constant (≈ 1), independent of p, while the weight range is {1,...,p}. No amount of structural cleverness can bridge this gap within the fiber induction framework.

## ATTACK 6: Weight Flexibility Induction Is Circular

The proposed strengthening: prove "zero-sums exist at f(k) distinct weights" when you have α(r)+k vectors.

To use this inductively, you need:
1. **Base case**: Weight Flexibility at rank 3 ✓ (Shadow Window Theorem, A8-specific)
2. **Inductive step**: Weight Flexibility at rank r−1 ⟹ Weight Flexibility at rank r ✗

The inductive step requires EXACTLY the same fiber decomposition + cross-fiber combination that the original problem does. The strengthened hypothesis doesn't make the inductive step easier — it makes it HARDER (you need to prove more).

More precisely: to prove "many achievable weights at rank r," you project to rank r−1 and try to combine weights from fibers. But Attack 3 shows: each fiber has surplus ≈ 1, giving O(1) weights. Combining O(1) sets via Cauchy-Davenport still gives O(1). You can't bootstrap O(1) to Ω(p).

**The circularity**: Weight Flexibility at rank r needs Weight Flexibility at rank r with k > 1 (to get more weights per fiber), which needs Weight Flexibility at rank r with k > 2, etc. The induction never grounds out.

## NEW FINDING: The Hard Core Is a Prescribed-Weight Sumset Problem

Every approach eventually reduces to the same bottleneck:

> Given n non-proportional vectors in F_p^r forming a ZSF set of size near α(r), and a target t ∈ F_p^r, prove there exist coefficients s_i ∈ {0,...,p-1} with Σs_i·v_i = t and **Σs_i = w for some prescribed weight w**.

This is the **prescribed-weight representation problem**. It's strictly harder than the unconstrained representation problem (which follows from standard counting arguments).

At rank 3, this is proved via the Shadow Window Theorem + character sums. At rank 4+, no tool handles it.

**The obstruction is not combinatorial but analytic**: the weight constraint Σs_i = w is an INTEGER constraint on the coefficients, not a modular one. Standard algebraic tools (CW, Nullstellensatz, character sums) work modulo p. The integer constraint lives outside their reach.
